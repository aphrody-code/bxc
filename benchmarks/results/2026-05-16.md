## Scenario: static-simple

| Scenario | Runner | p50 (ms) | p95 (ms) | Mean (ms) | Peak RAM (MB) | Success | Total (ms) |
|----------|--------|----------|----------|-----------|---------------|---------|------------|
| static-simple | bunlight-static | 3 | 4 | 3 | 62.4 | 100% | 2689 |
| static-simple | fetch-native | 1 | 1 | 1 | 67.5 | 100% | 2668 |
| static-simple | cheerio | 2 | 4 | 3 | 103.3 | 100% | 2762 |
| static-simple | jsdom | 22 | 35 | 25 | 260.4 | 100% | 4976 |

**Interpretation**: All runners operate on pre-fetched static HTML.
bunlight-static uses an in-process DOM transport (no spawn, no WebSocket).
fetch-native is the raw HTTP baseline. cheerio and jsdom add parsing overhead.
For pure static HTML, the fastest option is the one closest to raw fetch.

## Scenario: spa-react

| Scenario | Runner | p50 (ms) | p95 (ms) | Mean (ms) | Peak RAM (MB) | Success | Total (ms) |
|----------|--------|----------|----------|-----------|---------------|---------|------------|
| spa-react | bunlight-static | 6 | 10 | 6 | 256 | 100% | 1639 |
| spa-react | bunlight-fast | 30 | 39 | 30 | 262.2 | 100% | 3192 |
| spa-react | fetch-native | 1 | 3 | 1 | 264.3 | 100% | 1565 |

**Interpretation**: SPAs require JS execution to render content.
bunlight-static and fetch-native return the HTML skeleton only (~500 bytes).
bunlight-fast (Lightpanda) executes the page JS and returns the rendered content.
The latency difference reflects Lightpanda process spawn + JS execution overhead.

## Methodology

- All benchmarks use a **local mock HTTP server** (Bun.serve on :9999)
- This avoids rate-limiting, flaky networks, and spam of real websites
- Latency includes network RTT to localhost, HTML parse, and any JS execution
- RAM is measured via `process.memoryUsage().rss` (Bun process + any sub-processes)
- First run = cold start (includes warmup costs); subsequent runs = warm
- p50/p95 computed across all runs for a given runner/scenario pair
- Puppeteer/playwright runners are skipped if Chromium is not installed
- bunlight-fast runner is skipped if lightpanda binary is not in PATH or BUNLIGHT_LIGHTPANDA_BIN

## Reproducing

```bash
# Full suite
bun run benchmark

# Single scenario
bun benchmarks/run-all.ts --scenario static-simple

# With Lightpanda (fast profile)
BUNLIGHT_LIGHTPANDA_BIN=/path/to/lightpanda bun benchmarks/run-all.ts --scenario spa-react
```

## Honest caveats

- Benchmarks run against **localhost** — real-world latency will be dominated by network RTT
- Cloudflare bypass rates are from docs/PROFILES.md estimates, not measured in this suite
  (measuring requires real Cloudflare-protected URLs, risking rate limits)
- bunlight-fast (Lightpanda) cold start includes process spawn; warm-path is ~50-100 ms
- Chromium runners were not measured due to missing binary in test environment
  (add PUPPETEER_EXECUTABLE_PATH to include them)
- jsdom is not available in this environment (no node_modules/jsdom)
- All fast-profile SPA metrics come from docs/PROFILE-FAST-RESULTS.md
